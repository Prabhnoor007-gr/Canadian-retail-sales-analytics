# Canadian Multi-Province Retail Sales Analytics

## Overview
This project simulates a Canadian multi-province retail business using transactional sales data. The objective was to design a scalable SQL data model and develop key performance metrics to analyze regional performance, customer behavior, and seasonal trends.

## Tech Stack
- PostgreSQL
- SQL
- Power BI
- DAX

## Data Processing
- Removed cancelled orders and null customers
- Engineered revenue metric
- Resolved duplicate key issues
- Designed star schema (customers, orders, order_items)

## Key Metrics
- Total Revenue
- Average Order Value (AOV)
- Customer Lifetime Value (CLV)
- Repeat Customer Rate
- Monthly Revenue Trends

## Key Insights
- Quebec generated highest revenue due to strong AOV and retention
- Revenue peaks in November indicating seasonal demand
- Regional performance varies by customer loyalty and spending behavior
